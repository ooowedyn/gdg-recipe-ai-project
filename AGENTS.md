# 에이전트 작업 규칙

## 1. 전체 작업 방식

이 프로젝트는 3개의 기능 단위로 나누어 작업한다.

1. 식재료 인식 에이전트
2. 레시피 추천 에이전트
3. 레시피 시각화 에이전트

각 담당자는 자신의 폴더를 중심으로 작업한다.

## 2. 담당 폴더

### ingredient-agent

담당 기능:

- 사진 속 식재료 인식
- 인식된 식재료 리스트 생성

입력:

- 식재료가 포함된 이미지

출력:

- 식재료 이름 리스트

### recipe-agent

담당 기능:

- 식재료 리스트를 바탕으로 만들 수 있는 한식 메뉴 추천

입력:

- 식재료 리스트

출력:

- 추천 메뉴 목록

### visual-agent

담당 기능:

- 선택된 메뉴의 레시피를 단계별로 시각화

입력:

- 선택된 메뉴 정보
- 레시피 상세 정보

출력:

- 단계별 조리 과정

## 3. 공통 데이터 흐름

사진 입력
→ ingredient-agent
→ 식재료 리스트
→ recipe-agent
→ 추천 메뉴 목록
→ visual-agent
→ 단계별 레시피

## 4. 작업 규칙

1. 각 팀원은 자신의 담당 폴더를 우선 수정한다.
2. 공통 데이터 형식은 docs/data-format.md를 따른다.
3. 공통 프롬프트 규칙은 shared/prompt-rules.md를 따른다.
4. 입력과 출력 형식이 바뀌면 docs/data-format.md를 먼저 수정한다.
5. 실제 API 키는 GitHub에 올리지 않는다.
6. .env 파일은 개인 로컬에서만 사용한다.
7. GitHub에는 .env.example만 올린다.

## 5. 에이전트별 목표

### ingredient-agent 목표

이미지를 분석하여 사진 속 주요 식재료를 식재료 이름 리스트로 변환한다.

최종 출력 예시:

- 계란
- 대파
- 양파

### recipe-agent 목표

식재료 리스트를 보고, 현재 가진 재료만으로 만들 수 있는 한식 메뉴를 3개 이상 추천한다.

최종 출력 예시:

- 대파계란덮밥
- 계란국
- 양파계란볶음

### visual-agent 목표

추천된 메뉴의 조리 과정을 초보자도 이해하기 쉽게 단계별로 정리한다.

최종 출력 예시:

1. 재료 준비
2. 재료 손질
3. 조리 시작
4. 마무리
