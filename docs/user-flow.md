# 사용자 흐름

## 1. 사용자 상황

사용자는 냉장고에 식재료가 있지만 어떤 요리를 만들 수 있는지 바로 떠올리지 못한다. 직접 유튜브나 검색창에 재료명을 입력하는 대신, 앱을 통해 현재 가진 재료로 만들 수 있는 요리를 추천받고자 한다.

## 2. 기본 사용자 흐름

1. 사용자가 앱을 실행한다.
2. 식재료 사진을 촬영하거나 이미지를 업로드한다.
3. 앱이 사진 속 식재료를 인식한다.
4. 인식된 식재료가 리스트로 표시된다.
5. 사용자는 식재료 리스트를 확인한다.
6. 잘못 인식된 재료가 있으면 삭제하거나 수정한다.
7. 누락된 재료가 있으면 직접 추가한다.
8. 사용자는 최종 식재료 리스트를 확정한다.
9. 앱은 해당 재료만으로 만들 수 있는 한식 메뉴를 추천한다.
10. 사용자는 추천 메뉴 중 하나를 선택한다.
11. 앱은 선택한 메뉴의 조리 과정을 단계별로 보여준다.
12. 사용자는 단계별 레시피를 보며 요리를 진행한다.

## 3. 핵심 흐름 요약

사진 입력
→ 식재료 인식
→ 식재료 리스트 확인
→ 메뉴 추천
→ 레시피 상세 확인
→ 요리 진행

## 4. 예시 시나리오

사용자가 냉장고에서 계란, 대파, 양파를 꺼내 사진을 찍는다.

앱은 사진을 분석하여 계란, 대파, 양파를 인식한다.

사용자는 인식된 재료 리스트를 확인하고, 잘못된 재료가 없으면 메뉴 추천 버튼을 누른다.

앱은 대파계란덮밥, 계란국, 양파계란볶음과 같은 한식 메뉴를 추천한다.

사용자가 대파계란덮밥을 선택하면, 앱은 필요한 재료와 조리 과정을 단계별로 보여준다.

## 5. MVP에서 제외하는 흐름

다음 기능은 MVP 이후 확장 기능으로 분리한다.

- 냉장고 실시간 스캔
- 사용자 취향 기반 추천
- 보유 양념 자동 반영
- 조리 도구 자동 반영
- 유통기한 관리
- 장보기 목록 생성
