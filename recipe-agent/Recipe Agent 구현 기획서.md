# Recipe Agent 구현 기획서

## 1. 개요

`recipe-agent`는 인식된 식재료 리스트를 바탕으로 만들 수 있는 한식 메뉴를 추천하고, 각 메뉴의 조리 과정까지 정리하는 에이전트이다.
전체 흐름에서 두 번째 단계이며, `ingredient-agent`의 결과를 받아 레시피 정보를 생성한다.

## 2. 담당 범위

식재료 리스트 입력 -> 메뉴 추천 및 조리 과정 출력

## 3. 입출력 형식

**입력**

```json
{
  "ingredients": ["계란", "대파", "돼지고기"]
}
```

**출력**

```text
1. 돼지고기대파볶음
이름: 돼지고기대파볶음
난이도: 쉬움
시간: 20분
사용 재료: 돼지고기, 대파
추천 이유: 가진 재료를 활용하기 좋고 조리 과정이 간단하다.
조리 과정:
1단계: 돼지고기를 먹기 좋은 크기로 자른다.
2단계: 대파를 어슷하게 썬다.
3단계: 팬에 식용유를 두르고 돼지고기를 볶는다.
4단계: 대파와 간장을 넣고 함께 볶는다.
5단계: 접시에 담아 완성한다.
```

## 4. 사용 기술

- 언어: Python
- 모델: GPT-5.5
- 방식: API 연결 없이 GPT-5.5에 프롬프트를 직접 입력하는 방식

## 5. 구현 방식

`sample-input.json`에 식재료 리스트를 입력한 뒤 `run.bat`을 실행하면, GPT-5.5에 넣을 프롬프트가 자동으로 생성된다.

생성된 프롬프트는 `generated-gpt55-prompt.md`에 저장되고, 클립보드에도 복사된다.
이 프롬프트를 GPT-5.5 채팅창에 붙여넣으면 레시피 추천 결과를 받을 수 있다.

**처리 흐름**

```text
식재료 리스트 입력
-> 프롬프트 생성
-> GPT-5.5에 붙여넣기
-> 메뉴 추천 결과 출력
```

## 6. 파일 구성

```text
recipe-agent-runnable/
```

## 7. 프롬프트 조건

- 입력된 재료를 최대한 활용한다.
- 한식 메뉴를 추천한다.
- 초보자도 만들 수 있는 메뉴를 우선 추천한다.
- 기본 양념은 있다고 가정한다.
- 부족한 재료가 많이 필요한 메뉴는 제외한다.
- 결과는 보기 좋게 정리한다.

## 8. 결과 활용

`recipe-agent`의 결과는 메뉴 추천뿐 아니라 조리 과정까지 포함한다.
따라서 별도 단계 없이도 사용자가 바로 따라 할 수 있는 레시피 형태로 활용할 수 있다.

예를 들어 추천 결과가 아래와 같다면,

```text
이름: 돼지고기대파볶음
난이도: 쉬움
시간: 20분
사용 재료: 돼지고기, 대파
추천 이유: 가진 재료를 활용하기 좋고 조리 과정이 간단하다.
```

## 9. 정리

`recipe-agent`는 식재료 인식 결과를 받아 만들 수 있는 메뉴를 추천하고, 각 메뉴의 조리 과정까지 제공하는 역할을 한다.
