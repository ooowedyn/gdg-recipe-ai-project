# visual-agent 전달용 정리

## 연결 흐름

```text
ingredient-agent
사진에서 식재료 인식
예: ["계란", "대파", "돼지고기"]

recipe-agent
식재료 리스트를 받아 만들 수 있는 메뉴 추천
예: 돼지고기대파볶음

visual-agent
recipe-agent가 추천한 메뉴를 받아 단계별 조리 과정 생성
```

## recipe-agent가 visual-agent에게 넘길 결과 형식

```text
이름: 돼지고기대파볶음
난이도: 쉬움
시간: 20분
사용 재료: 돼지고기, 대파
추천 이유: 현재 가진 재료를 활용하기 좋고 조리 과정이 간단합니다.
```

## visual-agent 입력 예시

```text
선택 메뉴: 돼지고기대파볶음
난이도: 쉬움
시간: 20분
사용 재료: 돼지고기, 대파
추천 이유: 현재 가진 재료를 활용하기 좋고 조리 과정이 간단합니다.
```

## visual-agent 프롬프트 예시

```text
너는 visual-agent야.

recipe-agent가 추천한 메뉴 정보를 입력받아,
요리 초보자도 따라 할 수 있도록 조리 과정을 단계별로 정리해줘.

입력:
선택 메뉴: 돼지고기대파볶음
난이도: 쉬움
시간: 20분
사용 재료: 돼지고기, 대파
추천 이유: 현재 가진 재료를 활용하기 좋고 조리 과정이 간단합니다.

출력 조건:
- 조리 과정을 5단계 이상으로 나눈다.
- 각 단계에는 단계 번호, 행동, 설명을 포함한다.
- 어려운 표현은 쓰지 않는다.
- 사용 재료와 기본 양념을 중심으로 설명한다.
- 요리 초보자가 바로 따라 할 수 있게 짧고 명확하게 작성한다.

출력 형식:
1단계
행동: 재료 손질하기
설명: 돼지고기는 먹기 좋은 크기로 자르고, 대파는 어슷하게 썬다.

2단계
행동: 팬 예열하기
설명: 팬에 식용유를 두르고 중불로 예열한다.
```

## 발표할 때 말할 문장

```text
recipe-agent는 ingredient-agent가 인식한 식재료 리스트를 받아 만들 수 있는 한식 메뉴를 추천합니다.

출력 결과에는 메뉴 이름, 난이도, 조리 시간, 사용 재료, 추천 이유가 포함됩니다.

이 결과는 visual-agent로 전달되고, visual-agent는 선택된 메뉴의 조리 과정을 단계별로 정리하거나 시각화합니다.
```
